---

##  Key Insights

* The machine is part of an **Active Directory domain**
* MSSQL exposure is likely the main attack vector
* WinRM suggests a path for **post-exploitation access**
* Web application may provide initial foothold or credentials

---

##  Attack Path Hypothesis

1. Enumerate web application (port 80)
2. Obtain credentials (from web or elsewhere)
3. Access MSSQL (port 1433)
4. Extract hashes / sensitive data
5. Crack password or reuse credentials
6. Access system via WinRM (port 5985)
7. Privilege escalation to Administrator

##  Notes

* Presence of `DC01` strongly suggests Domain Controller
* Time skew detected (~7 hours) → may affect Kerberos attacks
* Focus should be on credential-based attacks rather than exploits
