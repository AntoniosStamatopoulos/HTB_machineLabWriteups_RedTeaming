# Pterodactyl HTB Lab

## Overview
This lab simulates a realistic attack scenario against a Linux system hosting a web application environment.  
It focuses on identifying weaknesses across different layers of the system and understanding how multiple issues can be chained together to achieve full compromise.

---

## Description
The environment includes a publicly accessible web service backed by a database and system-level components commonly found in modern Linux infrastructures.  
Participants are required to analyze the system, interact with exposed services, and leverage misconfigurations or vulnerabilities to progress from limited access to full system control.

The lab emphasizes the importance of understanding how different technologies interact, including web applications, system services, and access control mechanisms.

---

## Learning Outcomes
This lab helps develop skills in:

- System and service enumeration  
- Identifying and analyzing attack surfaces  
- Understanding web application behavior and backend integration  
- Working with Linux environments and permissions  
- Recognizing privilege boundaries and escalation opportunities  
- Thinking in terms of chained exploitation rather than isolated vulnerabilities  

---

## Skills Developed

### Technical Skills
- Linux system navigation and analysis  
- Web application interaction and inspection  
- Basic database interaction  
- Understanding authentication and access control mechanisms  

### Analytical Skills
- Problem-solving in complex environments  
- Correlating information from different sources  
- Identifying weak points in system design  
- Adapting strategy based on findings  

---

## Tools and Technologies
Participants may use a variety of tools depending on their approach, including:
